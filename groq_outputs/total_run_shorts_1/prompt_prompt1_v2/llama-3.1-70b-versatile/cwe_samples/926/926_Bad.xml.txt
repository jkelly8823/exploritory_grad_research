Exposed Intent Filter and Service Vulnerability

EXPLANATION:

However, without additional verification, services that don't use permissions can be exploited by any application. Furthermore, using the `DEFAULT` category allows other apps to invoke the activity using an implicit intent, which may lead to the escalation of privileges. RECOMMENDATIONS:
 
- Restrict the ability to launch these components by setting the `exported` attribute for both components to `false` if they should not be reached from external apps.
- Use explicit intents instead of implicit intents whenever possible to mitigate permission escalation risks.
- If an intent filter is required, define the required permission to restrict the ability to start the component. This is especially relevant to the service component.
