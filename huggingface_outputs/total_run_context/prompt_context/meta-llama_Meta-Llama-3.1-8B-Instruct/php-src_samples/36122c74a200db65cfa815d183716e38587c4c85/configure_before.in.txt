CVE Description:
Session fixation vulnerability in the Sessions subsystem in PHP before 5.5.2 allows remote attackers to hijack web sessions by specifying a session ID.  The vulnerability is caused by the fact that the session ID is not properly regenerated when a user logs in or out.  This allows an attacker to fixate a session ID on a user's browser and then hijack the session when the user logs in or out.  The vulnerability can be exploited by an attacker who has access to the user's browser.  The vulnerability is not present in PHP versions
