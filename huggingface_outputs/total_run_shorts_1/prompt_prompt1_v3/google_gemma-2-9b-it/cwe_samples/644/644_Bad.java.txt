

VULNERABLE: YES
VULNERABILITY_DESCRIPTION:  Cross-Site Scripting (XSS)
EXPLANATION: The code directly adds untrusted user input (`untrustedRawInputData`) to a HTTP response header (`response.addHeader(HEADER_NAME, untrustedRawInputData)`). This is a classic XSS vulnerability. An attacker could inject malicious JavaScript code into the `untrustedRawInputData`, which would then be executed in the browser of any user who receives the response
